- Stack tecnologico planeado:
- SwiftUI para la interfaz.
- SwiftData para persistencia local.
- Vision OCR para captura y reconocimiento de texto.
- Swift Charts para visualizacion de datos.

- Estrategia de producto:
- Offline-first como criterio principal.
- La app debe priorizar funcionamiento local sin red.
- La sincronizacion futura no debe bloquear el uso base.
- La persistencia local es obligatoria desde el inicio.

- Estrategia de desarrollo:
- Desarrollo por Vertical Slices.
- Cada entrega debe cerrar una capacidad funcional completa.
- Evitar construir capas abstractas sin una necesidad real.
- Priorizar iteraciones pequenas, testeables y acumulativas.

- Uso de Codex como agente:
- Codex se usa para implementar, refactorizar y documentar.
- Cada cambio debe terminar con validacion tecnica.
- Las tareas deben definir alcance y restricciones con claridad.
- El agente debe respetar la arquitectura acordada.
